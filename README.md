# Create-a-Strong-Password-and-Evaluate-Its-Strength-
Understand what makes a password strong and evaluate password strength using online passwordstrength checker tools.
## Objective
The objective of this task is to understand what makes a password strong and evaluate password strength using online password strength checking tools.

## Tools Used
- PasswordMeter (https://passwordmeter.com)
- Web Browser (Chrome/Edge)
- Internet Connection

## Task Description
In this task, multiple passwords with different complexity levels were created and tested using an online password strength checker. The results were analyzed to understand how password length, character variety, and complexity affect password security.

## Passwords Tested

| Password | Strength |
|-----------|-----------|
| password123 | Weak |
| Welcome2026 | Medium |
| Ganesh@2026 | Strong |
| G@n3sh#Cyber$2026 | Very Strong |
| VmG@2026!Secure#Lab | Extremely Strong |

## Evaluation Results

### Password 1: password123
- Contains only lowercase letters and numbers
- Easy to guess
- Vulnerable to dictionary attacks
- Weak security

### Password 2: Welcome2026
- Contains uppercase, lowercase, and numbers
- Better than simple passwords
- Moderate protection

### Password 3: Ganesh@2026
- Includes special characters
- Improved complexity
- Strong password

### Password 4: G@n3sh#Cyber$2026
- Mix of uppercase, lowercase, numbers, and symbols
- High entropy
- Very difficult to crack

### Password 5: VmG@2026!Secure#Lab
- Long length
- Multiple character types
- Extremely strong password

## Password Strength Factors

### Length
Longer passwords require significantly more time to crack.

### Complexity
Using a combination of:
- Uppercase letters
- Lowercase letters
- Numbers
- Special characters

increases security.

### Unpredictability
Avoid common words, names, dates, and keyboard patterns.

## Common Password Attacks

### Brute Force Attack
An attacker tries every possible combination until the correct password is found.

### Dictionary Attack
Uses a list of common words and leaked passwords to guess passwords.

### Credential Stuffing
Uses stolen usernames and passwords from previous data breaches.

### Phishing
Tricks users into revealing passwords through fake websites or emails.

## Best Practices Learned

- Use passwords with at least 12–16 characters.
- Include uppercase and lowercase letters.
- Include numbers and symbols.
- Avoid personal information.
- Use unique passwords for every account.
- Enable Multi-Factor Authentication (MFA).
- Store passwords in a password manager.
- Use passphrases when possible.

## Findings
- Password length greatly improves security.
- Special characters increase password complexity.
- Common words are vulnerable to dictionary attacks.
- Strong passwords significantly reduce the risk of unauthorized access.
- Multi-factor authentication adds an extra layer of protection.
## Conclusion
This task helped in understanding password security principles and evaluating password strength using online tools. Strong passwords should be long, unique, complex, and combined with multi-factor authentication for maximum security.
